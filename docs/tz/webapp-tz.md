## 1. Общий обзор платформы

NMNH Platform — единая финтех веб‑платформа для экосистемы трейдинг‑ментора No Money No Honey.
Объединяет публичный лендинг, личный кабинет ученика, панель ментора и реалтайм‑ленту торговых
сигналов в одном продукте уровня Binance/Bybit по качеству интерфейса.

### 1.1 Ключевые принципы дизайна
- Тёмная тема с неоновыми акцентами (cyan `#0AFFE0`, gold `#FFD700`).
- Микроанимации: skeleton loading, лайв‑счётчики, hover‑эффекты.
- GSAP для лендинга (3D, глитч, parallax), Framer Motion для кабинета.
- Mobile‑first: bottom nav на мобайле, sidebar на десктопе, PWA.
- Полный реалтайм: WebSocket + SSE + push‑уведомления.
- Tailwind CSS + shadcn/ui — design system.
- TradingView Lightweight Charts (PnL/статистика) + Multicharts виджет.

### 1.2 Роли и доступ

| Роль | Доступ | Авторизация |
|---|---|---|
| Гость | Лендинг, калькулятор, TV‑виджет, лидерборд, лента (размытая) | Нет |
| Ученик | Полный кабинет, сигналы с расчётом, аналитика, чат | WEEX UID → бот → JWT |
| Ментор | Вся платформа + панель управления + статус системы | Отдельные credentials |

## 4. Система авторизации

### 4.1 Поток авторизации ученика
1. `nmnh.io/login` → ввод WEEX UID → `[Войти через WEEX UID]`.
2. `POST /api/auth/request-code { weex_uid }`.
3. FastAPI проверяет UID через аффилиат API WEEX.
4. Если найден — 6‑значный код отправляется в Telegram через бота.
5. Ввод кода → `POST /api/auth/verify { weex_uid, code }`.
6. FastAPI возвращает JWT access + refresh; JWT в httpOnly cookie; редирект на `/app/dashboard`.

> 🔧 Шаги 4 и 5 — это **контракт между веб‑платформой и ботом** (общий код доставляет ученику).
> Описан в [unified-core.md → Контракт auth‑flow](../architecture/unified-core.md#контракт-auth-flow-uid--код).

### 4.2 JWT токены

| Токен | TTL | Хранение | Назначение |
|---|---|---|---|
| Access | 15 мин | httpOnly cookie | Авторизация запросов |
| Refresh | 30 дней | httpOnly cookie | Обновление access |

### 4.3 Защита роутов
- Middleware Next.js проверяет JWT на `/app/*` и `/admin/*`.
- Истёк access → авто‑refresh без перезагрузки; истёк refresh → редирект на `/login`.
- `/admin/*` — отдельная проверка роли `admin` в JWT.
- Rate limiting на `/api/auth/*`: ≤10 попыток / 15 мин / IP.

## 5. Публичный лендинг (`nmnh.io`)

Главная точка входа, визуальный уровень Apple.com.

### 5.1 Навигация (Header)
- Логотип NMNH (SVG, глитч при hover, GSAP).
- Центр: О проекте · Сигналы · Калькулятор · Лидерборд · FAQ.
- Справа: язык `[RU|EN]` + тема `[🌙|☀️]` + `[Войти]`.
- При скролле: backdrop‑blur. Мобайл: hamburger → slide‑in.
- Кнопка `[Войти]`: outline → cyan‑заливка при hover; если залогинен → `[Кабинет →]`.

### 5.2 Hero
- H1 «Торгуй как профи. Учись у лучших.» — посимвольное появление (GSAP TextPlugin).
- Подзаголовок про персональные сигналы.
- 3 CTA: `[⚡ Начать обучение]` → `/login`; `[📊 Открыть счёт на WEEX]` → аффилиат; `[📢 Telegram]`.
- Фон: Three.js/Spline (куб/частицы) + parallax + периодический глитч на заголовке.
- Лайв‑счётчики (из БД, анимация CountUp): сигналов отправлено, активных учеников, винрейт %,
  «Работаем с 2023».

### 5.3 «Как это работает»
4 шага с иконками (регистрация WEEX → бот → сигналы под депозит → торговля). Появление карточек по
скроллу (GSAP ScrollTrigger, задержка 0.2 c), hover — поднятие + неоновая тень.

### 5.4 Публичная лента сигналов
- Последние 5 сигналов в карточках: тикер, направление, плечо, время.
- Детали (цены/расчёты) размыты `blur(8px)` + оверлей «Войдите чтобы увидеть расчёт» + `[Войти…]`.
- Бейдж 🟢 АКТИВЕН / ⚫ ЗАКРЫТ. Новый сигнал: slide‑down + flash.

### 5.5 TradingView мультичарт
2×2 (BTCUSDT, ETHUSDT, XRPUSDT + выбор), тёмная тема, TF 15m, переключатель 1/мультичарт, на мобайле
1 график со свайпом.

### 5.6 Калькулятор позиции (публичный)

**Поля ввода:**

| Поле | Тип | Дефолт | Описание |
|---|---|---|---|
| Депозит ($) | number | 1000 | Размер депозита |
| Режим | toggle | Умеренный | Умеренный/Турбо |
| Плечо | slider 1–400 | 10 | Умеренный ≤25, турбо ≤400 |
| Пара | select/text | BTCUSDT | Торговая пара |
| Цена входа ($) | number | Авто с WEEX | Подтягивается |
| Стоп‑лосс ($) | number | Авто | Или % от входа |
| TP1/TP2/TP3 ($) | number | Авто | TP2/TP3 опционально |

**Вывод (реалтайм):** Маржа · Объём (маржа×плечо) · Риск $ (красный если >5%) · Риск % · Профит
TP1/2/3 (зелёный) · RR (бейдж 1:1.5…).

> 🔧 Формулы калькулятора — **тот же общий пакет `calculator`**, что в боте (единый источник истины).
> На клиенте — зеркальная реализация/обёртка над эндпоинтом `/api/market/calculate` (A‑03, A‑09).

**UX:** плавная смена при переключении режима; авто‑ограничение слайдера; `[🔄 Обновить цену]`;
предупреждение ⚠️ при риске >10%; `[Поделиться расчётом]` (shareable ссылка); `[Открыть в WEEX]`.

### 5.7 Статистика платформы (публичная)
6 карточек: всего сигналов · винрейт % · средний RR · лучший сигнал · активных учеников · объём
торгов учеников $. Анимация CountTo, ISR каждые 5 мин.

### 5.8 Лидерборд (ISR 60 сек)

### 5.11 Соцсети

### 7.3 Лента сигналов (`/app/signals`)
- Фильтры: статус `[Все/Активные/Закрытые]`, режим, период, поиск по тикеру 🔍.

  пользователя; цены (вход/текущая/стоп/TP1‑3 с дистанциями); прогресс‑бар с маркерами SL/Вход/TP;
  персональный расчёт (маржа/позиция/риск/профиты/RR); кнопки `[🚀 Войти в сделку]` `[📊 Детали]`

### 7.4 Детальная страница сигнала (`/app/signals/[id]`)

### 7.5 Аналитика (`/app/analytics`)
- **KPI:** Общий PnL · Винрейт (круговой прогресс) · средний профит/убыток · лучший/худший сигнал ·
  средний RR · серия побед 🔥.
- **График PnL** (TV Lightweight): линия баланса, точки сделок (зелёные/красные), переключатель

- **Серия сделок:** тепловая лента последних 50 (квадратики, tooltip).
- **Разбивка по тикерам:** bar (топ‑10 по объёму) + pie (распределение прибыли).
- **История сигналов:** таблица (20/стр), сортировка, фильтры, экспорт PDF/CSV.

### 7.6 Лидерборд (`/app/leaderboard`)

- **Мини‑статистика:** получено/активных/закрытых, мой PnL, позиция в лидерборде.
- **Экспорт:** PDF / CSV.
- `[Выйти]` (красная, с подтверждением).

## 8. Панель ментора (`/admin`)

Закрытая зона, доступ только ментору.

### 8.1 Навигация (sidebar, всегда виден)
Лого + `ADMIN`; `[📊 Дашборд][📡 Новый сигнал][📋 Сигналы][👥 Ученики][📈 Аналитика][💬 Чат]
[🖥️ Статус системы][⚙️ Настройки][📥 Экспорт]`.

### 8.2 Главный дашборд
- **KPI (5):** активных учеников · сигналов сегодня · активных сигналов · общий PnL учеников ·
  общий винрейт.
- **2 графика:** рост аудитории + сигналов по дням (bar).
- **Heatmap активности** учеников по дням (как GitHub).
- **Activity feed** (реалтайм): новый ученик / сигнал отправлен / сигнал закрыт / ошибка.
- **Лучшие сигналы:** топ‑3 по RR за месяц.

### 8.3 Форма нового сигнала (`/admin/signal/new`)
- **Rich‑text ввод** в свободном формате (как в TG) с подсказками; `[📎 Прикрепить фото]` (drag&drop,
  preview, удаление).
- `[🔍 Разобрать сигнал]` → **превью разобранного** (таблица: тикер/направление/плечо/вход/стоп/TP1‑3/
  тип маржи/тип входа) с бейджами `НАЙДЕНО` (синий) / `АВТО` (красный), каждую строку можно
  редактировать инлайн.
- **Выбор аудитории:** `[👥 Всем (N)][📊 Умеренным (N)][⚡ Турбо (N)]`.
- **Расчёт под аудиторию:** список учеников с балансом/режимом/маржой/объёмом/риском.
- **Кнопки:** `[✅ Отправить][👁️ Превью сообщения][❌ Отменить]`. Превью — модалка с табами
  Умеренный/Турбо (точный вид как в Telegram).
- **После отправки:** отчёт доставки (✅/❌ по каждому), `[Закрыть]` и `[Повторить для неудавшихся]`.

> 🔧 Парсер сигнала и шаблоны сообщений здесь — **те же**, что в боте (общий `core`). Никакого
> отдельного парсера для веба (A‑03, A‑05).

### 8.4 История сигналов (`/admin/signals`)
Таблица + пагинация, колонки Дата/Тикер/Направление/Плечо/Режим/Отправлено/Статус/RR/Действия,
фильтры, `[📡 Новый сигнал]`. Клик → детальная.

### 8.5 Управление учениками (`/admin/students`)
- Таблица: # · Аватар · Ник/TG · WEEX UID · Режим · Баланс · PnL · Активен · Действия
  (`[⚙️ Настроить][🔄 Обновить баланс][❌ Отключить]`), поиск, фильтр статуса.
- Клик → мини‑график баланса (30д) или `/admin/students/[id]` (полная карточка: профиль, график,
  история сигналов с PnL, статистика, `[Изменить режим/риск][Отключить][Удалить]`).
- **Ожидающие подтверждения:** список заявок `[✅ Принять][❌ Отклонить]`.

### 8.6 Аналитика сигналов (`/admin/analytics`)
KPI (всего/прибыльных/убыточных/без стопа/средний RR); график винрейта по времени; сигналов по дням;
heatmap тикеров; лучший/худший по RR; сравнение Умеренный vs Турбо; топ‑10 пар; pie LONG/SHORT.

### 8.7 Статус системы (`/admin/status`)

| Компонент | Индикатор | Детали |
|---|---|---|

## 9. Реалтайм архитектура

### 9.1 WebSocket
`ws://api.nmnh.io/ws`, аутентификация JWT в query, heartbeat 30 c, автореконнект (backoff 1/2/4/8 c).

### 9.2 События (сервер → клиент)

| Событие | Payload | Действие |
|---|---|---|
| `new_signal` | `{signal_data, calc_for_user}` | Карточка + баннер + push |
| `price_update` | `{symbol, price}` | Обновить цену |
| `signal_closed` | `{signal_id, result}` | Обновить статус + результат |
| `balance_update` | `{balance_usdt}` | Обновить баланс |
| `chat_message` | `{user, text, time}` | Сообщение в чат |
| `leaderboard_update` | `{positions}` | Перестановка |
| `system_status` | `{component, status}` | Статус в панели ментора |

### 9.3 SSE
Лёгкие односторонние обновления без авторизации: публичные цены и лайв‑счётчики на лендинге.

> 🔧 **Источник цен (A‑12):** `price_update` и публичные цены поступают от **единого серверного
> сборщика цен** (следит только за символами активных сигналов, троттлинг). Клиенты не подключаются к
> бирже напрямую. См. [unified-core.md → Сборщик цен](../architecture/unified-core.md#сборщик-цен-a-12).

## 13. Безопасность

| Угроза | Защита | Реализация |
|---|---|---|
| XSS | CSP | Next.js headers |
| CSRF | SameSite + CSRF token | FastAPI middleware |
| SQL Injection | Parameterized queries | SQLAlchemy ORM |
| Brute force | Rate limiting + lockout | 10/15 мин |
| DDoS | Rate limiting по IP | FastAPI + nginx |
| JWT кража | httpOnly + short TTL | 15 мин access |
| Утечка данных | HTTPS everywhere | SSL Render |
| Несанк. `/admin` | Проверка роли в JWT | Middleware |
| Инъекция в сигнал | Санитизация | FastAPI validators |
| Аудит | Логирование запросов | FastAPI middleware |

## 15. FastAPI — эндпоинты

> Единое ядро. Бот использует те же сервисы (calculator, weex, signals) внутри процесса или через
> внутренний API. См. [unified-core.md](../architecture/unified-core.md).

`POST /api/signals` (ментор) · `PATCH /api/signals/{id}/close` (ментор).

**Students (ментор):** `GET /api/students` · `GET/PATCH /api/students/{id}` ·
`POST /api/students/{id}/approve` · `DELETE /api/students/{id}` · `GET .../balance` · `GET .../analytics`.

**Analytics/Stats:** `GET /api/stats/public` · `/stats/leaderboard` · `/analytics/signals` (ментор) ·
`/analytics/me`.

**Chat:** `GET/POST /api/chat/messages` · `DELETE /api/chat/messages/{id}` (ментор) ·
`POST .../{id}/pin` · `WS /ws/chat`.

**Profile:** `GET/PATCH /api/profile` · `POST /api/profile/avatar` · `GET /api/profile/export/pdf|csv`.

**System (ментор):** `GET /api/system/status` · `GET/DELETE /api/system/logs` ·
`GET/PATCH /api/system/settings` · `POST /api/system/maintenance`.

**Price & Market:** `GET /api/market/price/{symbol}` · `GET /api/market/calculate` · `WS /ws/prices`.
